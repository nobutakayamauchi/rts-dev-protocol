# rts-dev-protocol

Structure-first development protocol for AI-operable business systems, with commander, meter, and modular split-ready architecture.

Status: PARTS / DEV-PROTOCOL / MINIMUM ALIVE

rts-dev-protocol is the development protocol shelf for the RTS ecosystem.

It holds reusable development rules, AI implementation contracts, PR hygiene notes, review checklists, and repository rescue procedures.

It is not RTS core.

It is not RTS-AGE.

It is not a runtime implementation repository.

It is not a product repository.

## Purpose

The purpose of this repository is to keep development procedure material separate from product, runtime, protocol, and component repositories.

Use this repository for:

- AI implementation contracts
- Codex task boundaries
- PR review checklists
- documentation-only rescue patterns
- repository classification workflows
- safe iteration procedures
- rollback and review notes

Do not use this repository for:

- product code
- runtime code
- canonical RTS protocol records
- secrets or credentials
- connector implementation
- business delivery workflows

## Current Priority

The next priority is to create an inventory of reusable development protocols and decide which ones are ready, draft, stale, duplicate, risky, or belong elsewhere.
